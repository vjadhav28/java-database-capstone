**Title:** As an admin, I want to manage the platform securely so that I can maintain control over the system.
**Acceptance Criteria:**
1. I can log into the portal with my username and password.
2. I can log out of the portal to protect system access.
3. I can add doctors to the portal.

**Priority:** High
**Story Points:** 5
**Notes:**
- Ensure secure authentication and authorization mechanisms are in place.
- Provide clear instructions for adding and managing doctor profiles.
-Delete doctor's profile from the portal
-Run a stored procedure in MySQL CLI to get the number of appointments per month and track usage statistics

**Title:** As a patient, I want to explore doctor options and manage my appointments so that I can receive timely healthcare.
**Acceptance Criteria:**
1. I can view a list of doctors without logging in to explore options before registering.
2. I can sign up using my email and password to book appointments.
3. I can log into the portal to manage my bookings.
4. I can log out of the portal to secure my account.

**Priority:** Medium
**Story Points:** 8
**Notes:**
- Ensure the user interface is intuitive and provides clear information about each doctor.
- Implement a robust booking system that prevents double-booking and allows for easy rescheduling or cancellation.
**Title:** As a patient, I want to log in and book an hour-long appointment to consult with a doctor so that I can receive timely healthcare.
**Acceptance Criteria:**
1. I can log into the portal with my username and password.
2. I can browse available doctors and their schedules.
3. I can select a time slot and book an appointment.
**Priority:** Medium
**Story Points:** 5
**Notes:**
- Ensure the booking process is smooth and provides confirmation of the appointment.
- Implement a notification system to remind patients of their upcoming appointments.

**Title:** As a patient, I want to view my upcoming appointments so that I can prepare accordingly.
**Acceptance Criteria:**
1. I can log into the portal with my username and password.
2. I can view a list of my upcoming appointments.
3. I can see details about each appointment, including date, time, and doctor information.
**Priority:** Medium
**Story Points:** 3
**Notes:**
- Ensure the appointment list is easy to navigate and provides all necessary information.
- Implement a filtering or sorting option to help patients find specific appointments quickly.

**Title:** As a doctor, I want to manage my appointments and patient information so that I can provide effective healthcare.
**Acceptance Criteria:**
1. I can log into the portal to access my appointment management features.
2. I can log out of the portal to protect my data.
3. I can view my appointment calendar to stay organized.
4. I can mark my unavailability to inform patients of only the available slots.
**Priority:** Medium
**Story Points:** 5
**Notes:**
- Ensure the doctor interface is intuitive and provides all necessary information for managing appointments and patient interactions.
- Implement a robust system for updating and maintaining doctor profiles with accurate specialization and contact information.
- Provide clear notifications for upcoming appointments and any changes to scheduled meetings.
